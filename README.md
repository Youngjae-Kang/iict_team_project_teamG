[G조] Focus Out: 빛나는 착함을 기록합니다

"조회수가 곧 선행인 세상, 당신은 찍을 것인가 구할 것인가?"

📸 보여주기식 선행과 💓 보이지 않는 진심 사이에서 갈등하는 인터랙티브 비주얼 노벨 프로젝트

1. 프로젝트 개요 (Project Overview)
**'Focus Out'**은 SNS의 '좋아요'와 '팔로워' 수가 인간의 도덕성을 평가하는 척도가 된 가상의 디스토피아를 배경으로 합니다. 플레이어는 일상 속에서 발생하는 위급 상황(길 잃은 아이, 심정지 환자, 선로 추락 사고)을 마주하게 되며, 매 순간 두 가지 선택지 앞에 놓입니다.

Camera ON: 상황을 촬영하여 SNS에 올려 '팔로워(사회적 평판)'를 얻는다. (팔로워 UP/ 낮은 구조가능성)

Camera OFF: 카메라를 끄고 직접 몸을 던져 사람을 구한다. (팔로워 보상 없음, 숨겨진 선행 점수 획득)

이 게임은 플레이어의 선택에 따라 달라지는 엔딩을 통해, **"진정한 선행이란 무엇인가?"**라는 윤리적 질문을 던집니다.

2. 주요 기능 및 특징 (Key Features)
📖 데이터 기반 스토리텔링 (Data-Driven Narrative)
JSON 기반 구조: 모든 대사, 선택지, 엔딩 조건이 data.json 파일에 구조화되어 있어, 코드 수정 없이 시나리오 확장이 가능합니다.

몰입형 연출: 상황에 맞는 배경 이미지 전환, 타이핑 효과, BGM 및 효과음으로 몰입감을 높였습니다.

🎮 선택에 따른 멀티 미니게임 (Multi-Gameplay)
플레이어의 선택(촬영 vs 구조)에 따라 전혀 다른 장르의 미니게임이 진행됩니다.

📸 FOCUS (촬영 모드): 흔들리는 초점을 조절하여 결정적인 순간을 포착하는 타이밍 게임.

🏃 CROSSY (구조 모드 - Ep1): 빗길 횡단보도에서 차량을 피해 아이에게 도달하는 아케이드 게임.

🧊 ICE & 🚑 CPR (구조 모드 - Ep2): 미끄러운 복도를 지나 제세동기를 가져오는 아케이드 게임 + 제세동기를 클릭하는 타이밍 게임

🎣 FISHING (구조 모드 - Ep3): 물리 엔진이 적용된 낚시 메커니즘을 활용하여 선로에 떨어진 사람을 끌어올리는 구조 게임.

📱 인스타그램 UI 시스템
에피소드가 끝날 때마다 가상의 인스타그램 피드 화면이 출력됩니다.

플레이어의 성과에 따라 '좋아요' 수와 '팔로워' 수가 실시간으로 갱신되어, 시각적 보상과 심리적 갈등을 유발합니다.

🔚 멀티 엔딩 시스템 (Multi-Ending)
게임 내내 누적된 두 가지 변수 **scoreLikes (보여주기식 점수)**와 **scoreHidden (양심 점수)**의 조합에 따라 3가지 엔딩으로 분기됩니다.

Bad Ending: 껍데기뿐인 영웅 (팔로워는 많으나 진심은 없음)

Normal Ending: 평범한 시민 (적당한 타협)

True Ending: 무명의 성자 (기록되지 않았지만 세상을 바꿈)

3. 에피소드 구성 (Episode Guide)
Ep 1,"비 오는 날, 차도에 방치된 아이" , Focus(사진 촬영) , Crossy(길 건너기)
Ep 2,교내 응급 상황,심정지로 쓰러진 친구 , Focus(브이로그 촬영) , Ice Run & CPR(장애물 달리기 + 심폐소생술)
Ep 3, 선로에 떨어진 취객,  Focus(영웅적 구조 촬영) , Rescue(Fishing)(물리 기반 구조)

4. 기술적 구현 (Technical Implementation)
🏗️ Finite State Machine (유한 상태 머신)
게임의 흐름을 gameState 변수로 관리하여 체계적인 화면 전환을 구현했습니다.

MAIN_MENU → INTRO → TUTORIAL → SCENE → CHOICE → RULEBOOK → MINIGAME → RESULT → INSTAGRAM → ENDING

🛠️ 모듈화 된 코드 구조
sketch.js: 게임의 메인 루프(draw, setup) 및 입력 처리 (keyPressed, mousePressed)

story.js: JSON 파싱, 대화창 출력, 타이핑 효과 구현

minigames.js: 5가지 미니게임의 개별 로직 및 물리 연산 포함

ui.js: 인스타그램 화면, 타이틀, 튜토리얼 등 UI 렌더링 담당

variables.js: 전역 변수 및 에셋(이미지, 폰트) 관리

⚙️ 주요 알고리즘
물리 엔진 (Physics): FISHING 게임에서 velocity, gravity, thrust를 활용해 낚시 바의 관성 움직임을 구현.

충돌 처리 (Collision): ICE 게임에서 사각형(AABB) 충돌 및 반사각(vel *= -1.2)을 계산하여 통통 튀는 타격감 구현.

반응형 UI: 튜토리얼 및 룰북에서 사용자의 입력에 따라 동적으로 변하는 가이드 UI 구현.
